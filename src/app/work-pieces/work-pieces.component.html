Roller

The main premise for the project is to demonstrate the simplicity of implementing a post-processor that can be used as an imperative accessibility feature. Utilising C++ and UE4 Blueprints, I was able to create a post-processor that renders the textures of assets that is in the path of the viewport. In this case, the post-processor accounted for individuals with colour weaknesses pertaining to eyesight. The options available address dichromatic diseases deuteranopia and protanopia - dichromatic diseases usually affecting red or green retinal photoreceptors.

Dish Pig Dilemma

Created in a small team of 5 people, the project was ultimately discarded, but not without some well-learned lessons. The project involved the use of Git in a UE4 project, which taught me valuable real-world skills down the track. At that time, none of us had used Git before and we had experienced nothing but horror at pushing our projects to and fro. The mechanics I developed in this game was simple: the player was to use both joysticks on either an Xbox or PS4 controller and the player had to pick up the dishes that were spawning next ot the sink, wash them in the sink by dunking it in the water, and placing them in the rack for points. The confronting mechanic was that the controls were intentionally inverted, to give the player a hard time as it was no longer intuitive. This project taught me software development essential skills like Git, communication and time management. As well as key game development skills like kinematics and skeleton animations.

Flipping Time!

I really enjoyed working on this game mechanic demonstration. Using lists in C# and .NET, I was able to to store key variables and recall them in reverse order. I used a method that would record transform positions, quaternions and velocity. I then created two other methods – one is called per frame so long as the player held the backspace key down and another that would ‘rewind’ the player by calling the function that was loop through the variable from the last state that the recording method stored last and work its way backward. I optimised the recording system by flushing out the rewind capabilities according to the maximum amount of time the player can rewind. This demo is part of a 2D platform game called Flipping Time! It gives the player the opportunity to complete the level should they fall over by rewinding themselves back to the point of fault. There is a bar that fills and is proportional to the amount of time they’re able to rewind.

The player must break through, shoot or tunnel through layers of meteors in order to get to the end. This game had a team of 5 people with me as the project manager.
